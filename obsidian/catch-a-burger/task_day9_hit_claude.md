# Задача: исполнение удара сковородой (HitResolver + сигналы)

> Контекст — в `CLAUDE.md`. ИЗОЛИРОВАННАЯ задача.
> Часть кода разработчик пишет САМ (см. «Что уже сделано вручную»).
> Твоя зона — только то, что в «Требования». Раздел ГРАНИЦЫ соблюдать строго.

---

## Что нужно сделать

Повар бьёт сковородой по своей стороне → отбивает ближайший к устью ингредиент
в зоне досягаемости. Отбитый ингредиент исчезает (не пойман, не упал — жизнь не
теряется). Удар в пустоту — промах (ничего не сбито, но это нормально).

Механика идёт через сигнал (симметрично ловле): повар стреляет `ChefHitSignal`,
отдельный резолвер исполняет удар по ленте.

---

## Что УЖЕ сделано вручную разработчиком (НЕ трогать, использовать как есть)

**1. `ChefController` уже стреляет `ChefHitSignal`** при ударе (с кулдауном).
Ты `ChefController` НЕ трогаешь.

**2. В `ConveyorSystem` уже добавлен метод (готовая сигнатура, ВЫЗЫВАЙ его):**

```csharp
// Бьёт ближайший к устью ингредиент на стороне side в зоне досягаемости
// (Progress >= minProgress). Возвращает true + тип через out, если попал.
// false — промах (в зоне никого).
public bool TryHit(Side side, float minProgress, out IngredientType hitType)
```

`Side` и `IngredientType` — в namespace `BurgerCatch.Gameplay.Conveyor`.

---

## Архитектурный контекст

Unity 6 + Zenject. Событийная архитектура через SignalBus. Системы — не
MonoBehaviour, биндятся в контейнере. Паттерн: `sealed`, `IInitializable` +
`IDisposable`, подписка в `Initialize()`, отписка в `Dispose()`, зависимости
через конструктор. Сигналы — `sealed`, иммутабельные.

---

## Требования

### 1. Сигналы (namespace `BurgerCatch.Events`)

- `ChefHitSignal` — несёт `Side Side` (на какой стороне ударил повар).
  (Его УЖЕ стреляет ChefController — ты только создаёшь класс сигнала, если его ещё нет.)
- `IngredientHitSignal` — несёт `IngredientType Type` и `Side Side`. Стреляется,
  когда удар реально что-то сбил (для будущих эффектов: звук, частицы, полёт).

### 2. Класс `HitResolver` (namespace `BurgerCatch.Gameplay.Conveyor`)

- `sealed`, реализует `IInitializable`, `IDisposable`.
- Зависимости через конструктор: `SignalBus`, `ConveyorSystem`.
- Константа `MinHitProgress = 0.6f` — порог зоны досягаемости (бьём только то,
  что уже близко к устью). Позже уедет в конфиг.
- В `Initialize()`: подписаться на `ChefHitSignal`.
- В `Dispose()`: отписаться.
- На `ChefHitSignal`: вызвать `_conveyor.TryHit(signal.Side, MinHitProgress, out var type)`.
  - Если вернул `true` (попал) → выстрелить `IngredientHitSignal(type, signal.Side)`.
  - Если `false` (промах) → ничего не делать (удар в пустоту — норма, кулдаун уже
    потрачен на стороне ChefController).

### 3. Регистрация в `GameplayInstaller`

Добавить ТОЛЬКО:
```csharp
Container.DeclareSignal<ChefHitSignal>();          // если ещё не задекларирован
Container.DeclareSignal<IngredientHitSignal>().OptionalSubscriber();
Container.BindInterfacesAndSelfTo<HitResolver>().AsSingle();
```
Если `ChefHitSignal` уже задекларирован разработчиком — не дублировать декларацию.

## ГРАНИЦЫ — что НЕ делать

- **НЕ трогать `ChefController`** — он уже стреляет `ChefHitSignal` с кулдауном.
- **НЕ трогать `ConveyorSystem`** кроме ВЫЗОВА готового метода `TryHit`. Не менять
  его, не добавлять туда логику удара — она в `HitResolver`.
- **НЕ реализовывать** эффекты удара (отскок, полёт в камеру, slow-mo, частицы,
  звук) — это Фаза 5. Сейчас отбитый просто исчезает (это делает `TryHit`).
- **НЕ трогать** ловлю (`CatchResolver`), заказ (`OrderSystem`), стек, цену, жизни.
- **НЕ менять** правило: отбитый ингредиент НЕ теряет жизнь и НЕ идёт в бургер.
  Он просто удаляется (это уже делает `TryHit`). `HitResolver` только сигналит факт.
- **НЕ добавлять** пакеты, не менять настройки.

## Как проверить (критерий приёмки)

Временные логи в `_Sandbox`:
1. Повар на стороне Left, на ленте Left есть ингредиент близко к устью (Progress
   >= 0.6) → тап по Left (повар уже там) → ингредиент исчезает, в консоли
   `IngredientHit` с его типом. Жизнь НЕ теряется, в бургер НЕ идёт.
2. Удар, когда на стороне нет ингредиента в зоне (пусто или все далеко вверху) →
   промах, ничего не сбито, `IngredientHit` НЕ стреляется. Это нормально.
3. Спам ударами невозможен — кулдаун (его держит ChefController, не твоя забота,
   но в тесте видно, что удары не чаще ~раз в 0.3 сек).
